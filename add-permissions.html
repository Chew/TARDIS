Add permissions

If you are using a permissions manager that has per world config files, you would normally need to manually add permissions for players when they create a TARDIS in its own world. By setting add_perms: true in the TARDIS config file, you can get the plugin to automatically add a permissions group with your desired permissions attached to it.

To do this:

  1. With TARDIS installed, run the server once so that it creates its default files
  2. Stop the server and edit plugins/TARDIS/permissions.txt
  3. Use the example (in file and) below:

Example permissions.txt

tardis_users
tardis.use
tardis.room
tardis.timetravel.player
tardis.timetravel.location
-tardis.exile
  • The first line is the name of the permissions group the player should be added to, so it would be something like: Builder or Donor
  • Then it's just one permission per line (any permission, not just TARDIS ones)
  • Add a minus in front of the permissions to make it false.

This feature supports Essentials GroupManager and bPermissions only at this stage.
