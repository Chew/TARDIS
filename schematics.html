Schematics

You can now use any schematic file you want to create the inner TARDIS, but there are a few things you need to know before you jump right in and start replacing files in the TARDIS plugin folder.

Guidelines for TARDIS schematics

  1. You need 1 file per size. Get the names of the files from the TARDIS plugin folder... they should be ‘budget.schematic’, ‘bigger.schematic’, and ‘deluxe.schematic’
  2. The TARDISes can only have 4 repeaters, 1 iron door, 1 stone button, 1 wood button, and 1 sign in their initial designs.
  3. TARDISes can be any size, and the chunks they use are reserved and cannot be used by other TARDISes - but be aware that big TARDISs will lag the server!
  4. TARDISes should be squares i.e. that same width (x) and length (z) dimensions. A 16 x 16 block TARDIS is good, a 24 x 18 block TARDIS would be bad.
  5. When you are facing the outside of the TARDIS door, you should be pointing SOUTH.
  6. The handbrake should be set as a CAKE block so that the plugin knows to record the loaction as the handbrake.

Making the schematics

The WorldEdit plugin was used to create the default set of TARDIS schematics.

So you could:

  1. Build a TARDIS
  2. Make a cuboid WorldEdit selection e.g. with //hpos1 and //hpos2
  3. //copy
  4. //schematic save [size] - replacing [size] with either Deluxe, Bigger or Budget
  5. Move the new schematic files into the plugins/TARDIS/schematics folder and start up your server.
