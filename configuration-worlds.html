World configuration

These configuration options relate to the plugins use of worlds.

Create worlds

Set whether the (inner) TARDIS is constructed in its own world.

Setting this option to true requires that the TARDISChunkGenerator plugin and a multi-world plugin (preferably Multivers-Core) are also enabled on the server. If either of these options is unavailable, create_worlds will be automatically set to false.

create_worlds: [true|false]

The default is true.

Game mode

Set which game mode TARDIS worlds are set to upon creation.

gamemode: [survival|creative]

The default is survival.

Inventory group

If the Multiverse-Inventories plugin is enabled on the server, this option sets the world group TARDIS worlds are automatically added to when they are created. If you don’t wish to use this feature (even if Multiverse-Inventories is installed) set it to 0

inventory_group: [world group name|0]

The default is 0.

Permissions

If the server uses a permissions plugin with per-world config files (such as Essentials GroupManager), then you can automatically set the player's permissions for TARDIS worlds.

add_perms: [true|false]

The default is true.

To set up the permissions that the player receives, edit the permissions.txt file in the plugins/TARDIS folder. For more information on setting this up visit the Add permissions page.

Default world

Set whether the (inner) TARDIS is constructed in a specific world (for instance a FLAT world).

default_world: [true|false]

The default is false.

Default world name

Set the default world that the TARDIS is constructed in when default_world is true.

default_world_name: [World]

The default is myridiculouslylongworldnameiscalledcuthbert

Include default world

Toggle whether the default world is included in the possible destinations when time traveling in the TARDIS.

include_default_world: [true|false]

The default is false.

The End

Whether travel to End worlds should be enabled for the plugin

the_end: [true|false]

The default is false.

The Nether

Set whether NETHER worlds are included in random Time Travel destinations

nether: [true|false]

The default is false.

Worlds configuration section

The worlds section is a list of worlds on the server and whether they are enabled for time travel locations.

The format is:

worlds:
   [world]: [true|false]
   [world]: [true|false]
   ...

Worlds are added and removed automatically when the plugin is enabled - it will find newly added worlds, and remove worlds that no longer exist.

You should rarely need to edit this section, as worlds can be included or excluded using the /tardisadmin include [world] and /tardisadmin exclude [world] commands.
